# JAVA 开发规范

目标：

- 容易维护
- 健壮
- 可复用

  分类 | 指导
  :---|:---
  编码 | UTF-8  
  注释 | 支持Swagger
  git模式| PR(pull-request) / merge request
  代码静态检查 | 遵守风格规范
  代码提交| 必须通过review
  测试| 所有代码都有单元测试
  集成测试| 所有代码必须通过集成测试才能提交到分支或主干  


  [编码风格](https://google.github.io/styleguide/javaguide.html)
